# xhs-poster Public API for LLM Agents

This app accepts external draft content through a small token-protected API.
Use these endpoints when an AI assistant needs to add post material to xhs-poster.

## Base URL

```text
https://poster.movenotes.club
```

## Authentication

All `/api/public/*` endpoints require:

```http
Authorization: Bearer <PUBLIC_API_TOKEN>
```

Use `Content-Type: application/json` for JSON requests, and `multipart/form-data` for image uploads.
Never put the token in prompts that will be shared publicly. Ask the user for the token out of band.

## Health Check

```http
GET /api/public/health
```

Expected response:

```json
{ "success": true, "service": "xhs-poster-public-api" }
```

## Upload Image

```http
POST /api/public/upload
```

Uploads one image and returns a public URL that can be used in `images` when creating posts.
Use `multipart/form-data` with a `file` field. Only image files are accepted.

Request:

```bash
curl -X POST "https://poster.movenotes.club/api/public/upload" \
  -H "Authorization: Bearer $PUBLIC_API_TOKEN" \
  -F "file=@./cover.jpg"
```

Response:

```json
{
  "success": true,
  "url": "https://host.movenotes.club/xhs-poster/assets/1700000000000-id-cover.jpg",
  "key": "1700000000000-id-cover.jpg"
}
```

## Query Posts

```http
GET /api/public/posts
```

Returns existing posts. Optional query parameters:

- `status`: `draft`, `published`, or `all`; defaults to all statuses.
- `limit`: 1-100; defaults to 50.
- `q`: case-insensitive search over title and content.
- `category`: exact material category match. `tag` is still accepted for backward compatibility.

Request:

```bash
curl "https://poster.movenotes.club/api/public/posts?status=draft&limit=20" \
  -H "Authorization: Bearer $PUBLIC_API_TOKEN"
```

Response:

```json
{
  "success": true,
  "posts": [
    {
      "id": 123,
      "title": "标题",
      "content": "正文内容",
      "status": "draft",
      "images": ["https://host.movenotes.club/xhs-poster/assets/image.jpg"],
      "categories": ["小红书"]
    }
  ],
  "total": 1,
  "limit": 20
}
```

## Query One Post

```http
GET /api/public/posts/{id}
```

Returns one post with full image and category records.

Request:

```bash
curl "https://poster.movenotes.club/api/public/posts/123" \
  -H "Authorization: Bearer $PUBLIC_API_TOKEN"
```

## Create Draft Post

```http
POST /api/public/posts
```

Creates one draft post. `title` and `content` are required. `categories` and `images` are optional arrays.
Image URLs must be absolute `http://` or `https://` URLs.
If the image is a local file, call `/api/public/upload` first and use the returned `url`.
`tags` is still accepted as a backward-compatible alias for `categories`.

Request:

```json
{
  "title": "标题",
  "content": "正文内容",
  "categories": ["小红书", "AI"],
  "images": ["https://example.com/image.jpg"],
  "status": "draft"
}
```

Response:

```json
{
  "success": true,
  "created": [{ "index": 0, "postId": 123 }],
  "failed": []
}
```

## Create Multiple Draft Posts

```http
POST /api/public/posts
```

Batch mode accepts up to 20 posts per request:

```json
{
  "posts": [
    {
      "title": "第一篇标题",
      "content": "第一篇正文",
      "categories": ["分类1"],
      "images": []
    },
    {
      "title": "第二篇标题",
      "content": "第二篇正文",
      "categories": ["分类2"],
      "images": ["https://example.com/cover.png"]
    }
  ]
}
```

## cURL Example

```bash
IMAGE_URL=$(curl -fsS -X POST "https://poster.movenotes.club/api/public/upload" \
  -H "Authorization: Bearer $PUBLIC_API_TOKEN" \
  -F "file=@./cover.jpg" | jq -r '.url')

curl -X POST "https://poster.movenotes.club/api/public/posts" \
  -H "Authorization: Bearer $PUBLIC_API_TOKEN" \
  -H "Content-Type: application/json" \
  --data '{
    "title": "DePIN 节点部署笔记",
    "content": "整理一篇适合小红书发布的 DePIN 节点部署经验...",
    "categories": ["DePIN", "Web3", "节点"],
    "images": ["'"$IMAGE_URL"'"]
  }'
```

## Notes for AI Agents

- Prefer `status: "draft"` unless the user explicitly asks for `published`.
- Keep titles short and readable.
- Do not invent image URLs. Upload local image files first, then use the returned `url`.
- If a request partially fails, inspect the `failed` array and retry only those items.
